# Vibe Tools

Набор маленьких самостоятельных инструментов, каждый из которых — отдельная HTML-страница.

## Структура

- Каждый инструмент — отдельный `.html` файл, который можно открыть независимо.
- В будущем планируется общая страница-каталог со ссылками на все инструменты.

## Методология

Разработка ведётся по **TDD (Test-Driven Development)**:

1. **Red** — сначала пишем тест, который падает
2. **Green** — пишем минимальный код, чтобы тест прошёл
3. **Refactor** — улучшаем код, сохраняя зелёные тесты

Тесты располагаются рядом с инструментом или в отдельной папке `tests/`.

## Документация

Папка `docs/` содержит рабочие документы проекта, разбитые по подпапкам:

```
docs/
  catalog/        — главная страница-каталог
    spec.md
    plan.md
    logs.md
  image-text/     — надпись на картинке
    spec.md
    plan.md
    logs.md
```

Каждый инструмент (и главная страница) имеет свою папку с файлами:

- `spec.md` — спецификация (создаётся по запросу перед разработкой)
- `plan.md` — план имплементации (создаётся после спецификации)
- `logs.md` — лог важной информации по ходу имплементации

## Именование

Каждый инструмент — отдельный `.html` файл с коротким, ёмким названием из одного слова, без дефисов. Например: `stamp.html`, не `image-text.html`.

## Инструменты

### 1. stamp — Надпись на картинке

Инструмент для наложения удобно читаемой текстовой надписи на изображение. Результат можно сохранить и использовать, например, как пост в Telegram-канале.
